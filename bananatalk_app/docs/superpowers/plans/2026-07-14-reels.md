# Reels (Language-Learning Short Video) Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A Reels tab in Moments — thumbnail grid landing → full-screen vertical swipe feed of ≤180s language-learning videos — riding on the existing Moment model, with prompt-camera + free-form creation, reactive moderation (auto-hide at 2 reports + admin restore), Apple-1.2 content-policy gate, and a `REELS_ENABLED` kill switch.

**Architecture:** Reels are Moments with `isReel:true` (video upload/thumbnails/CDN/likes/comments/reports all inherited). New: a cursor-paginated `GET /moments/reels` endpoint with two-bucket soft language ranking; a controller-pool vertical pager built from the orphaned `VideoFeedItem`; report-threshold auto-hide wired into the `Report`-collection flow the admin panel reads; reels excluded from the regular card feeds.

**Tech Stack:** Node/Express + Mongoose (backend, node:test), Flutter + Riverpod + video_player/video_compress/video_trimmer (app), DigitalOcean Spaces + CDN.

### Task 2: Reels feed endpoint + kill switch + app-config

**Files:** Create `lib/reelsFeed.js` (extend), Modify `controllers/moments.js`, `routes/moments.js`, `config/limitations.js`, `controllers/appConfig.js`; Test `test/reelsFeed.test.js`.

**API contract (the app builds against this):**
`GET /api/v1/moments/reels?limit=10&before=<ISO createdAt cursor>` → `{ success, data: [moments (populated user, video, language, promptId, likeCount, commentCount, isLiked)], nextCursor: <ISO|null> }`. Gated by `REELS_ENABLED` (404 when off). Filters: `isReel:true, privacy:'public', hiddenPendingReview:{$ne:true}, isDeleted:{$ne:true}`, poster ∉ viewer's blocked set. Ranking: **two buckets** — A: `language` ∈ {viewer target ISO, viewer native ISO}; B: rest — each `createdAt` desc, concatenated A then B **per page window** (partition applied after the cursor window is fetched; document that ordering is bucket-stable within a page, cursor remains pure createdAt — simple and honest at this scale).

- [ ] **Step 1: Write failing tests** for the pure parts in `lib/reelsFeed.js`: `partitionByLanguage(reels, [targetIso, nativeIso])` returns A-then-B stable order; `buildReelsQuery({before, blockedIds})` builds the exact filter incl. cursor `createdAt: {$lt: before}`; kill-switch guard behavior (reuse the `roomsEnabledGuard`-style pattern from `lib/roomMembership.js`).
- [ ] **Step 2:** Run → fail. Implement helpers. Run → green.
- [ ] **Step 3:** Implement `getReelsFeed` in `controllers/moments.js` using the helpers + existing `getBlockedUserIds` (`moments.js:53`) + existing `toIso` for the viewer's languages. Wire `router.get('/reels', protect, reelsEnabledGuard, getReelsFeed)` in `routes/moments.js` — **register BEFORE any `/:id` route** so 'reels' isn't captured as an id.
- [ ] **Step 4:** `REELS_ENABLED` in `config/limitations.js` (same one-liner pattern as `ROOMS_ENABLED`); `reelsEnabled` added to the app-config response (`controllers/appConfig.js` next to `roomsEnabled:61`).
